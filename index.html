1.2 Creating a Structured Document

Concepts of a Website

Metaphor: Think of HTML as the structure of the house. CSS files are the style of the house, e.g. color of the walls. Java Script files are interactive components, they are changing some element of your house, e.g. garage door opener.
HTML is considered the language of the web with syntax and rules using tags into elements.
CSS use syntax and rules to change how elements look on the webpage

Tree Like Structure

Branching structure of the website consisting of elements. Also referenced as document object model (DOM) which is the standard convention for representing and interacting with elements in HTML.

Boxes

Consider every element as a "box" on the page. Make your life a lot easier by thinking in terms of "boxes".

Design Mockup

Left pane shows information of colors
How do I draw a box around every image element? Everything on the page are boxes. Start with big to small boxes. Big box on the full page. You can use pencil to draw the boxes, or even cut the draft into boxes.

1.3 Adding CSS Style and HTML Structure

Selector and Classes

Search and Replace - class or tag of element to property values
Order or inheritance matters - default styles of browser, stylesheet in a separate file, singlesheet style inside HTML, and inline style in an element.
Specifics matter - right type of element for the particular style
Selector - defines what elements of a page a particular style will apply

Helpful Hints

Avoid repetition by using CSS
Use sizing percentages of an element box instead of pixels
Set box-sizing to border-box for every element by using "*{box-sizing:border-box;}"
DIV are block elements. They take up entire width of page
Add style rule "display: flex;" to make DIVs appear next to each other
Add in HTML using "style" tag
Make sure to validate code using Markup Validation Service and CSS Validation Service
